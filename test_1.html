Как посмотреть историю запросов к базе данных PostgreSQL через консоль? 1#$%Чтобы просмотреть историю запросов к базе данных PostgreSQL через консоль, вы можете воспользоваться утилитой `pgBadger` или использовать встроенный журнал (`log`) PostgreSQL. 1. Используя pgBadger: - Установите pgBadger на свой компьютер. - Подключитесь к своей базе данных PostgreSQL через консоль. - Запустите команду для записи журнала в файл: ``` ALTER SYSTEM SET log_destination = "csvlog"; ALTER SYSTEM SET logging_collector = on; ALTER SYSTEM SET log_directory = "pg_log"; ALTER SYSTEM SET log_filename = "postgresql-%Y-%m-%d_%H%M%S.log"; ALTER SYSTEM SET log_statement = "all"; ``` - Перезапустите сервер PostgreSQL. - Выполните запросы к базе данных. - После выполнения запросов, запустите pgBadger, указав путь к журнальному файлу, чтобы создать отчет: ``` pgbadger /path/to/postgresql-X.log ``` - Откройте созданный отчет в браузере, чтобы просмотреть историю запросов. 2. Использование встроенного журнала: - Откройте файл `postgresql.conf`, обычно он находится в каталоге `/etc/postgresql/⟨version⟩/main/`. - Найдите и измените или добавьте следующие параметры: ``` log_statement = "all" logging_collector = on log_directory = "pg_log" log_filename = "postgresql-%Y-%m-%d_%H%M%S.log" ``` - Перезапустите сервер PostgreSQL. - Выполните запросы к базе данных. - Журнальные файлы будут сохранены в каталоге `pg_log`. - Вы можете открыть соответствующий файл журнала с помощью команды `less` или любого другого текстового редактора, чтобы просмотреть историю запросов. Обратите внимание, что использование журнала запросов может повлиять на производительность сервера PostgreSQL, поэтому рекомендуется активировать его только при необходимости. А как посмотреть историю подключения к порту сервера Ubuntu 20? 2#$%Чтобы просмотреть историю подключений к порту сервера Ubuntu 20, вы можете использовать утилиту `ss` (Socket Statistics) или команду `netstat`. Вот примеры: Используя `ss`: 1. Откройте терминал на сервере Ubuntu 20. 2. Выполните следующую команду для просмотра активных подключений к порту: ``` ss -ntlp sport = ⟨порт⟩ ``` Замените `⟨порт⟩` на конкретный номер порта, к которому вы хотите просмотреть историю подключений. Например, 80 для HTTP или 22 для SSH. 3. Вы увидите активные подключения к указанному порту на сервере. Используя `netstat`: 1. Откройте терминал на сервере Ubuntu 20. 2. Выполните следующую команду для просмотра активных подключений к порту: ``` netstat -ntlp | grep ⟨порт⟩ ``` Замените `⟨порт⟩` на конкретный номер порта, к которому вы хотите просмотреть историю подключений. Например, 80 для HTTP или 22 для SSH. 3. Вы увидите активные подключения к указанному порту на сервере. Обратите внимание, что для выполнения команд `ss` или `netstat` может потребоваться использование привилегий суперпользователя. Если у вас нет достаточных прав, добавьте `sudo` перед каждой командой для выполнения команд с правами суперпользователя. Журналы системы: - Откройте терминал на сервере Ubuntu 20. - Чтобы просмотреть историю подключений через SSH, выполните команду: ``` cat /var/log/auth.log | grep sshd ``` - Вы увидите список записей журнала, относящихся к подключениям через SSH. 2. Логи SSH: - Откройте терминал на сервере Ubuntu 20. - Чтобы просмотреть историю подключений через SSH, выполните команду: ``` cat /var/log/secure | grep sshd ``` - Это предполагает, что на вашем сервере используется файл логов `/var/log/secure` для записи информации о подключениях SSH. Если файл называется иначе, замените путь и имя файла в команде. Обратите внимание, что для просмотра логов системы или логов SSH может потребоваться использование привилегий суперпользователя. Если у вас нет достаточных прав, добавьте `sudo` перед каждой командой, чтобы выполнить их с правами суперпользователя.
